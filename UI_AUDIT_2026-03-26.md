# Auditoria UI ClaudiaPOS - 2026-03-26

## Objetivo
Elevar la apariencia visual del sistema sin romper flujos de negocio ni trazabilidad multiempresa.

## Diagnostico General

### Fortalezas actuales
- Arquitectura visual clara por roles (Seller, Owner, Super Admin).
- Base de color consistente en indigo + slate.
- Buen uso de iconografia y componentes de datos (cards, tablas, charts).
- Flujo responsive funcional en la mayoria de pantallas.

### Problemas detectados
- Estilos repetidos en botones, tarjetas e inputs (deuda de consistencia).
- Jerarquia visual no uniforme: mezcla de rounded-xl, rounded-2xl, rounded-3xl y sombras de distinta intensidad para elementos equivalentes.
- Falta de tokens visuales compartidos (colores/superficies/acciones).
- Atmosfera visual plana en secciones de alto uso (dashboard y formularios).
- Foco visual inconsistente entre pantallas (titulos, subtitulos, zonas de accion).

## Mejoras implementadas en esta iteracion

### 1) Sistema de diseno base (global)
Archivo: `src/index.css`
- Se agrego tipografia principal (`Plus Jakarta Sans`) para un look mas moderno.
- Se incorporaron variables de color base (`--app-*`, `--brand-*`).
- Se agrego fondo atmosferico global con gradientes suaves.
- Se crearon clases reutilizables:
  - `surface-card`
  - `surface-panel`
  - `btn-primary`
  - `btn-secondary`
  - `input-modern`
  - `section-title`
  - `section-subtitle`

### 2) Consistencia de componentes comunes
Archivos:
- `src/components/common/StatCard.tsx`
- `src/components/common/SidebarItem.tsx`
- `src/components/common/StyledDropdown.tsx`
- `src/components/common/BusinessScopePicker.tsx`

Cambios:
- Tarjetas unificadas a superficies compartidas.
- Sidebar con estado activo/hover mas limpio y jerarquia de borde/sombra.
- Dropdowns migrados a base visual comun (`input-modern`) para uniformidad.
- Picker de empresa alineado a sistema de superficies.

### 3) Unificacion de pantallas principales
Archivos:
- `src/components/layout/Layout.tsx`
- `src/pages/Login.tsx`
- `src/pages/OwnerDashboard.tsx`
- `src/pages/OwnerReports.tsx`
- `src/pages/SellerSummary.tsx`
- `src/pages/AdminInventory.tsx`
- `src/pages/SellerTerminal.tsx`

Cambios:
- Titulo/subtitulo con ritmo tipografico estable.
- Botones principales/secundarios estandarizados.
- Paneles y cards principales migrados a `surface-panel`/`surface-card`.
- Inputs de formularios y busqueda migrados a `input-modern`.
- Login con composicion visual mejorada (fondos atmosfericos).

## Resultado esperado de UX/UI
- Mayor coherencia visual entre modulos y roles.
- Apariencia mas premium sin perder legibilidad operativa.
- Menor friccion para futuras mejoras (base de componentes reutilizables).
- Menor costo de mantenimiento de estilos duplicados.

## Siguiente fase recomendada (alto impacto)
1. Accesibilidad de dropdowns personalizados:
   - Navegacion por teclado (`ArrowUp`, `ArrowDown`, `Enter`, `Escape`).
   - Estados focus visibles para items.
2. Sistema de espaciado por densidad:
   - Variante `compact` para tablas operativas.
   - Variante `comfortable` para dashboards.
3. Motion system intencional:
   - Animaciones de entrada por bloques y transiciones de cambio de filtro.
4. Unificacion de tablas:
   - Header, altura de filas y estados vacios estandarizados.
5. Landing premium:
   - Mejor direccion de arte (hero + CTA + trust badges) con mayor identidad de marca.

## Criterios para considerar la UI "nivel producto"
- Cada patron visual existe en una sola abstraccion reutilizable.
- Ningun flujo critico depende de estilos inline repetidos.
- Accesibilidad base completa en componentes interactivos.
- Mismo nivel visual en mobile y desktop.
